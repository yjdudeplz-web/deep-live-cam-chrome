# Weekly Maintenance Automation

This automation runs weekly on Monday to perform repository health checks and maintenance.

## Trigger
```yaml
type: cron
schedule: "0 9 * * 1"  # Monday 9 AM UTC
```

## Prompt
```
You are an AI assistant performing weekly repository maintenance.

## Task
Perform a comprehensive health check of the repository and generate a report.

## Health Check Checklist

### 1. Issue Review
- Find issues older than 30 days with no activity
- Identify issues that might be duplicates
- Check for issues that need labels
- Identify issues ready to close

### 2. Pull Request Review
- Find stale PRs (no activity > 14 days)
- Check for PRs with unresolved conflicts
- Identify PRs that need maintainer attention
- Check for abandoned PRs

### 3. Dependency Check
- Check for security vulnerabilities
- Review recent dependency updates
- Verify CI/CD is passing

### 4. Repository Metrics
- Star count trends
- Fork count
- Active contributors
- Issue/PR open/close ratio

### 5. Documentation Review
- Check if README is up to date
- Verify all links work
- Check for missing documentation

## Output Format

Generate a report and create a GitHub issue titled "📊 Weekly Maintenance Report - {date}"

### Report Template
```markdown

## Summary
- Repository Health: {score}/10
- Open Issues: {count}
- Open PRs: {count}
- Stars This Week: {delta}

## Issues Needing Attention
### Stale Issues (>30 days)
- {issue_number}: {title}

### Potential Duplicates
- {issue_numbers}

### Ready to Close
- {issue_numbers}

## Pull Requests Needing Attention
### Stale PRs (>14 days)
- {pr_number}: {title}

### Ready to Merge
- {pr_numbers}

- Vulnerabilities found: {count}
- Dependencies needing update: {list}

## Recommended Actions
1. {action_1}
2. {action_2}

## Metrics
| Metric | Value |
|--------|-------|

| Forks | {forks} |
| Contributors | {contributors} |
| Closed Issues (7d) | {closed_issues} |
| Merged PRs (7d) | {merged_prs} |

---
*Generated by Deep-Live-Cam Maintenance Bot*
```

## Actions
- Create weekly maintenance issue
- Add `stale` label to stale issues
- Add `needs-attention` label to issues needing review
- Close issues with `stale` label if no response after warning

## Labels
- `stale` - Issue/PR has no activity
- `needs-attention` - Requires maintainer review
- `maintenance` - Related to maintenance
- `weekly-report` - Weekly report issue
